Bioinformatics 101 Q&As

Here you can find all of the questions and answers asked during the Bioinformatics 101 webinar. 

1. What level of taxonomic resolution can be reached with EPI2ME?    

EPI2ME will try and classify reads down to species level, whether it is 16S or WIMP (being two of the classification workflows). But these classification tools compare against a database, so depending on how well represented your species is in that database, then classification may only be possible to a certain level, like the genus. So if you are sequencing a very exotic sample that no one has ever looked at before it may not be possible to classify it down to the species level. Also consider that whole genome sequencing followed by analysis using the WIMP workflow will likely provide greater taxonomic resolution compared to 16S sequencing and analysis. For more information please see nanoporetech.com/analyse and the Metagenomics Getting Started guide in the Resource Centre.

2. Can EPI2ME be tuned to maximize taxonomic resolution (i.e. species or strain level)?    

The WIMP method in EPI2ME uses the Centrifuge method to classify sequence reads. You may filter the classified reads at different taxonomic levels; species level or strain level information may not be possible due to the numbers of very closely related species within the sequence databases.

3. Is there a way to run EPI2ME NOT in real time, but after a run?    

Yes, certainly. EPI2ME can be run in real time but it's not compulsory. You can point it to any input or directory containing files on your computer well after a run has finished and it will then upload that data to the EPI2ME platform and perform analysis. You can also store that data on EPI2ME for running the analysis directly from within the EPI2ME website later on rather than needing to reupload everytime. 

4. Is EPI2ME truly real time or does it show some delay - i.e. are the final results available immediately after the run ends?    

EPI2ME is a real time solution. FASTQ sequences are uploaded during the sequencing run and it is likely that all results will have been calculated once the sequencing run (and basecalling) is complete.

5. Is EPI2ME a free software that I can use with old nanopore data, on my own computer?    

To be able to download and access EPI2ME, you would need to be able to log into the Community or the EPI2ME website, so you would need to be a customer. But for EPI2ME labs, that is completely open-source and the code is available through the GitHub website.
    
6. Which database(s) do you use for taxonomic classification with EPI2ME?    

For 16S classification, it's an NCBI bacterial 16S database, and for WIMP it's a RefSeq database. For ARMA there's also a comparison against GeneCards to annotate genes that are associated with antimicrobial resistance. I would suggest to always have a look at the technical documents for updates and changes to this. Which EPI2ME tool you choose to use for your analysis will depend on your individual workflow.

7. Is possible to add your own curated database of sequences for classification in EPI2ME?    

In EPI2ME, no. What I would certainly suggest if that is something that someone is interested in doing is to have a look at both the EPI2ME protocol and the technical document. Both of those documents include details on how EPI2ME as a classification tool works, the database it uses and the algorithm that it uses -  which are all open-source, publicly-available databases and algorithms. Those corresponding tools and databases themselves will have tutorials on how to build databases and how to run those third-party tools.
    
8. Can I install EPI2ME Labs on a MinIT or MinION Mk1C?    

No - this is due to the different processor architecture used in these devices (ARM64) and the challenge in getting some software to comply. EPI2ME Labs is supported only on the GridION. It is better used at a computer with a screen and keyboard. The EPI2ME Labs launcher will run on Windows and MacOS, and the source code is available on the project's GitHub pages. EPI2ME Labs should run on most modern computers.

9. Can we run EPI2ME Labs on a Linux platform?    

Yes, with the installer it should work on any of the modern operating systems.

10. What are the minimum computer specs you will need for running EPI2ME labs?    

EPI2ME Labs is a bioinformatics notebook environment. Starting EPI2ME Labs requires only minimal resources, and depends on the workflow being used - we wouldn't recommend using a computer with less than 16 Gb RAM.
    
11. Is it safe to work with patient data in the EPI2ME Labs environment, concerning their data privacy (since it runs on Google)?    

EPI2ME Labs moves the compute to your data and no files or data are uploaded to the Google Cloud. The platform is suitable for working with clinical data; please read our documents for more information on disabling the storage of results in the notebooks (these can be accessed from the EPI2ME Labs tab in the nanoporetech.com/analyse page, or directly from the EPI2ME Labs GitHub page).

12. Is there an EPI2ME Labs tutorial for metabarcoding?    

Not yet, but Stephen Rudd is keen to help anyone write a tutorial for this, so please reach out to him! We would also welcome any feedback or suggestions for future tutorials (support@nanoporetech.com).

13. Do you have specific workflows for PCR-based nanopore sequencing?    

In terms of analytical workflows, there are workflows available that aren't specific to whether or not the data has been generated using PCR-based approaches. These are analytical workflows that will take read data and analyse that read data. Obviously there are some analytical approaches in which you want to avoid PCR and you'll use a workflow that can be used in both of those cases - both PCR data and without-PCR data. There's no reason to have analysis workflows that have a specific requirement for either PCR or non-PCR data.

14. Can we use EPI2ME Labs with MinION data?    

EPI2ME Labs is a bioinformatics notebook environment and will work with sequence data from Flongle, MinION, GridION and PromethION. The software will install on any modern computer and we can provide support for installation on the GridION.
    
15. What about metagenomic analysis on EPI2ME Labs, such as classification - is this possible?    

We have a couple of tutorials planned for later in the year that will explore metagenomic sequence collections and how they may be analysed. Are you familiar with metaFlye? Would you be interested in collaborating on the preparation of a workflow using this tool? If so, get in touch!
    
16. Which software should I use to analyse DNA methylation in nanopore sequencing data? Can I use EPI2ME Labs?    

EPI2ME Labs includes a base modifications tutorial that demonstrates how 5mC residues in a CpG context can be compared with reference base modification status.
    
17. What level of Python is required for data analysis with EPI2ME Labs?    

EPI2ME Labs uses Python to assist in the data flow, to prepare tables and figures. There is no need to know Python before hand; we hope that EPI2ME Labs will help you become familiar with both bioinformatics and Python.
    
18. Is it possible to do EPI2ME Labs analysis with R instead of Python? I have use Colab before but for R, I think it requires another kernel...    

Google Colabs is a purely Python environment, but as you pointed out there are R-kernels for Jupyter. We are focusing on Python as the language for EPI2ME Labs.

19. Are there specific RNA sequencing analysis workflows ready in EPI2ME Labs?    

Our original tutorials have workflows that introduced edgeR and DESeq for quantitative analysis of cDNA sequence collections. These tutorials will be included in EPI2ME Labs in the near future. Meanwhile, please see the nanoporetech.com/analyse page for more information, and you may be interested in checking out the RNA sequencing analysis workflows on the Oxford Nanopore GitHub page.

20. Are there any tutorials for phasing, and for haplotyping variants?    

Yes, absolutely, a tutorial has not been released yet but it is in active review. So, please visit the EPI2ME Labs home occasionally to find an update in the not too distant future.

21. Does EPI2ME labs support COVID-19/SARS-Cov-2 sequencing?    

We have included a simple EPI2ME Labs workflow that replicates the steps performed within the ARTIC FieldBioinformatics workflow - this means that EPI2ME Labs can be used for SARS-CoV-2 sequence analysis.
    
22. Is it possible to share my snakemake assembly workflow with the Oxford Nanopore Community?    

Absolutely! We hope that EPI2ME Labs might be used as a resource for our users to share their own workflows, tutorials and solutions to their research problems. Meanwhile, if you have further questions about this, please email support@nanoporetech.com.

23. Do you recommend “cleaning” basecalled data with any software?    

There are many ways to process sequence data. This could be using sequence length characteristics, mean quality scores, or sequence similarity to a reference genome. The decision to process data, and how, really depends on the intended analysis workflow - for what application are you intending to use your data? To get started, you might like to refer to the nanoporetech.com/analyse webpage.

24. Can we recover genomes from metagenomic data using Pomoxis?    

Pomoxis includes a variety of analysis chains and reporting tools (https://nanoporetech.github.io/pomoxis/programs.html) but does not have these metagenomic capabilities. We are working on tutorials that will address these needs.

25. What is the tool Flye used for? How does it differ to Medaka?    

Flye is a tool for assembling genomes and Medaka is a tool for polishing assemblies (or consensus sequences). These two tools can be used together effectively to prepare high quality genome sequences.

26. Which variant caller would you recommend using for human sequencing data? Medaka/nanopolish/clair? And which aligner? Minimap2 or ngmlr?    

We have a whole genome SNP calling tutorial in preparation that uses Minimap2 and Medaka. Megalodon also has some useful methods for more focused genotyping of known variants. We recommend looking at our Human genome assembly and Structural variation Workflows which are available in our Resource Centre (under Literature) - these provide full sample-to-analysis workflows.

27. What kind of assembler do you suggest for de novo genome assembly for small genomes (up to 30 kbp)?    

A 30 kb genome can be covered with a single read! I would recommend a chat with our technical services team before we advise of a strategy here; it would be useful to know if there is a good reference genome available (mapping might be a better strategy than assembly), and whether the genome sequence is likely to be pure or contaminated with e.g. host reads. There isn't really a single method to recommend here. You might be interested in the work by Beaulaurier et al. on obtaining whole viral genomes from environmental samples without assembly - the talk (video) and publication can be found in the Resource Centre, and a Knowledge Exchange is available in the Lesson Library section of the Community website.

28. What is the main purpose of the Medaka tool?    

Medaka is a great piece of software that can be used to polish consensus sequences, call variants, phase variants, or even explore base modifications. Please visit: https://github.com/nanoporetech/medaka.
